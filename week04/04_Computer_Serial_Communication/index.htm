Directionality of Serial Communication

TX/RX:

Computer (TX) → Arduino (RX)

Bidirectional communication is possible:

Handle carefully to avoid collision

Example: PC sends L:1\n → Arduino turns on LED

Characteristics of the Serial Protocol

Baud rate:

e.g. 9600, 115200 (must match)

Data format:

1 start bit, 8 data bits, optional parity, 1 stop bit

Delimiters:

\n mark message ends, , to split messages

Parsing Commands on Arduino

Read data:

Serial.read(), readStringUntil('\n'), parseInt()

Design simple commands: L:1, M:100, B:ON

Advanced: Implement state machines for complex control

Interfaces to Send Commands

Processing:

myPort.write('message') for example "L:1\n"

Python:

pyserial Easy script-based control

TouchDesigner:

Serial DAT node

Max/MSP: serial object

Designing a Communication Protocol

  • Use human-readable ASCII for prototyping
  • Add headers, delimiters, and checks

  • Feedback mechanisms with messages such as:

    OK, ERR, VAL:xxx

Sync and Error Handling

Avoid overflow:

Don’t spam commands

Use acknowledgments: Wait for OK

Add timeouts and fallbacks

Safety and Robustness

Validate commands on Arduino before executing

Set limits on input values

Example: LED should not blink faster than the hardware can handle

Historical & Technological Context

  • RS-232 (1960s): First widespread serial standard
  • Used in: Mainframes, modems, teletypewriters
  • Legacy: Foundation of USB and embedded interfaces

Why Serial Is Still Relevant
